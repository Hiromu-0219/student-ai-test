# 生徒AI・教育シミュレーション進捗まとめ

作成日: 2026-07-13

## 目的

一次方程式を学習する生徒AIを使い、教師1人に対して複数生徒がいる授業場面をシミュレーションするためのMVPを作成している。

生徒AIの知識状態・誤概念・性格はLLM内部ではなくJSONで管理し、LLMは主に発話生成器として扱う方針。

## 現在できていること

- `S001`, `S002`, `S003` の複数生徒を用意
- 各生徒が同じ教師発話に対して反応する
- 授業中に観察できる情報だけを `observable_event` として抽出
- 伝達AIが複数生徒の発話をまとめて要約
- 教師側の生徒理解である `teacher_belief` を観察から更新
- 複数回授業を行い、`teacher_belief` の推定理解度とconfidenceの推移を表・グラフで確認
- クラス全体対応と個別対応を分けて授業方略を出力

## 設計上の重要点

生徒AIの真の内部状態と、教師が観察から推定する状態を分けた。

```text
true_student_state
  生徒AI内部の真値。教師AIには直接見せない。

observable_event
  授業中に見える情報。発話、回答、正誤、反応時間、質問有無、途中式有無など。

teacher_belief
  教師が日々の観察から持つ生徒理解。confidence付きで更新する。
```

これにより、初対面では生徒の特徴が分からず、授業を重ねることで徐々に見立てが蓄積される構造にした。

## 主な追加ファイル

- `notebooks/teaching_strategy_experiment.ipynb`
  - 複数生徒の授業反応、伝達AI要約、teacher_belief更新、授業方略出力をColab上で確認するノートブック
- `src/observer/observation_filter.py`
  - 生徒AIの内部状態を隠し、授業中に観察できる情報だけに変換
- `src/observer/trait_classifier.py`
  - 伝達AI。個別発話分類と3〜20人のクラス要約に対応
- `src/teacher/belief_manager.py`
  - 観察イベントから教師側の推定状態を更新
- `src/teacher/intervention_planner.py`
  - クラス全体対応と個別対応をルールベースで計画
- `data/teacher_beliefs/`
  - 教師ごとの生徒推定状態を保存
- `data/observations/`
  - 授業中の観察イベントログを保存

## 現在のAI利用状況

- 生徒AIの発話生成にはLLMを使える設計
- 現在のColab検証では軽量確認のため `USE_MOCK_MODEL = True` を主に使用
- 伝達AI、teacher_belief更新、授業方略計画は現時点ではLLMなしのルールベース
- 今後LLM教師やLLM伝達AIに置き換えられるよう、入出力形式は辞書・JSON中心にしている

## 現在の検証結果

- テストは `python -m pytest` で全件通過
- 現在の最終確認では `45 passed`
- ColabノートブックはJSONとして読み込み確認済み

## 次にやる候補

1. 授業方略を実際に次の教師発話へ反映する
2. クラス全体対応と個別対応を受けて、次の問題や声かけを生成する
3. 伝達AIまたは教師AIの一部にLLMを入れて、ルールベースとの差を比較する
4. 授業前後で生徒AIの真の知識状態を更新する仕組みを接続する

## 共有時の補足

現段階では、教育シミュレーション全体の完成版ではなく、以下の流れが動くことを確認した段階。

```text
複数生徒AIの反応
→ 観察可能情報への変換
→ 伝達AIによるクラス要約
→ 教師側belief更新
→ クラス全体・個別対応の計画
```

教師が生徒の内部パラメータを直接見るのではなく、授業中の観察から徐々に推定する構造にした点が今回の主な進展。
