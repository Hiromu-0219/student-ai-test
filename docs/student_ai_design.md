# 生徒AI 設計仕様書

作成日: 2026-07-23

## 目的

本仕様書は、教育シミュレーションに組み込む生徒AIの設計を固定するための文書である。

本プロジェクトの生徒AIは、人間学習者を完全に再現するものではない。目的は、教師AIの授業設計を検証するために、理解度、誤概念、個人特徴を制御できる「限定的な代理生徒」を構築することである。

## 本研究で扱う妥当性

本研究では、実際の生徒データとの比較による外的妥当性は扱わない。

扱うのは、教育シミュレーションに使う代理生徒としての内部妥当性である。

```text
内部妥当性:
生徒AIの内部状態を操作したとき、設計意図に沿った正答率、誤答傾向、発話特徴が一貫して得られるか。

外的妥当性:
生徒AI集団の振る舞いが、実際の生徒集団のテスト結果や授業中反応にどの程度近いか。
```

現時点では実クラスのテスト結果や授業中発話データを持たないため、外的妥当性は今後の課題として明記する。

論文での安全な主張は次である。

```text
本研究では、生徒AIを人間学習者の完全な再現としてではなく、
理解度・誤概念・個人特徴を明示的に制御できる代理学習者として設計する。
評価では、内部状態の操作に対して期待される反応が一貫して得られるかを検証する。
```

## 基本構成

生徒AIは次の3層で構成する。

```text
生徒AI = 認知モデル + 個人特徴モデル + LLM発話生成器
```

| 層 | 役割 | LLMに任せるか |
| --- | --- | --- |
| 認知モデル | 正答率、誤答、スキル弱点、誤概念の影響を制御する | 任せない |
| 個人特徴モデル | 自信、質問傾向、意欲、Big Fiveを発話スタイルへ変換する | 一部だけ任せる |
| LLM発話生成器 | 生徒らしい自然文を生成する | 任せる |

重要なのは、LLMに「その生徒が正解できるか」を自由に決めさせないことである。正誤や誤答傾向は外部状態と認知モデルで制御し、LLMはその結果を自然な発話にする。

## 生徒状態

生徒状態は `data/students/{student_id}.json` で管理する。

| フィールド | 役割 |
| --- | --- |
| `student_id` | 生徒ID |
| `name` | 表示名 |
| `understanding` | 単元ごとの大まかな理解状態 |
| `knowledge_state` | 単元・スキル別の理解度 |
| `error_tendency` | 誤答傾向 |
| `misconceptions` | 誤概念 |
| `learning_speed` | 学習の伸びやすさ |
| `personality` | 追加の性格記述 |
| `big_five` | Big Five |
| `self_efficacy` | 自己効力感 |
| `question_tendency` | 質問傾向 |
| `motivation` | モチベーション |
| `learning_history` | 学習履歴 |

知識状態は0-100のスコアで扱う。

```json
{
  "knowledge_state": {
    "linear_equation": {
      "score": 60,
      "can_solve_ax_plus_b_equals_c": 65,
      "can_transpose_terms": 55,
      "can_divide_by_coefficient": 45,
      "can_handle_negative_numbers": 50,
      "can_handle_fractions": 35
    }
  }
}
```

性格・心理系の段階評価は原則として次の5段階で扱う。

```text
very_low, low, medium, high, very_high
```

## 認知モデル

認知モデルは、生徒の真の内部状態から、問題に対する解答方針を決める。

主な出力:

| 出力 | 意味 |
| --- | --- |
| `target_correct` | この問題で正答させるか |
| `correct_probability` | 正答確率 |
| `target_answer` | 生徒AIが最終的に出す答え |
| `misconception_strength` | 誤概念の強さ |
| `active_misconceptions` | 今回の問題に効いている誤概念 |
| `rationale` | なぜその正誤になったか |

正答確率は、理解度そのものではなく、内部状態から観測される結果として扱う。

本設計では、教育シミュレーション・知的学習支援でよく使われる BKT（Bayesian Knowledge Tracing）と IRT（Item Response Theory）の考え方を簡略化して取り入れる。

```text
理解度・スキル習得度 = 生徒AIの潜在的な知識状態
正答・誤答 = 潜在状態、問題難易度、誤概念、guess/slip から生成される観測結果
```

そのため、理解度80の生徒が常に80%正解する、という扱いにはしない。理解度は正答確率を決める主要因だが、問題の難易度や必要スキルとの一致によって結果は変わる。

基本式は次のように考える。

```text
正答確率 =
  ability_skill_match
  + guess
  - slip
  - misconception_penalty
  + affective_adjustment
```

各要素の意味:

| 要素 | 意味 |
| --- | --- |
| `ability_skill_match` | 生徒の該当スキル習得度と、問題難易度の差から決まる基礎正答確率 |
| `guess` | 未習得でも偶然・部分理解で正答する可能性 |
| `slip` | 習得していても計算ミスや読み違いで誤答する可能性 |
| `misconception_penalty` | 問題に関連する誤概念がある場合の低下量 |
| `affective_adjustment` | 自己効力感、モチベーション、不安などによる小さな補正 |

BKT的には、各スキルの習得度を `P(learned)` に近い値として扱い、正答確率を次のように解釈する。

```text
P(correct) = P(learned) * (1 - slip) + (1 - P(learned)) * guess
```

IRT的には、生徒のスキル習得度と問題難易度の差を使う。

```text
P(correct) = sigmoid(skill_mastery - problem_difficulty)
```

実装上は、この2つを厳密な統計モデルとして推定するのではなく、教育シミュレーション用に解釈可能な近似モデルとして使う。

実装では、比較のために2種類の認知モデルを用意している。

| モデル | クラス | 役割 |
| --- | --- | --- |
| 従来モデル | `CognitiveModel` | 理解度・スキルスコアを中心に正答確率を決める |
| BKT/IRT寄りモデル | `BKTIRTCognitiveModel` | スキル習得度、問題難易度、guess、slipを加味して正答確率を決める |

比較実験は `compare_cognitive_models()` で実行できる。

```python
from src.experiment import (
    compare_cognitive_models,
    export_cognitive_model_comparison_for_codex,
)

result = compare_cognitive_models(
    student_id="S001",
    test_id="linear_equation_20q_001",
    understanding_levels=list(range(0, 101, 10)),
    use_mock_model=True,
)

export_cognitive_model_comparison_for_codex(result)
```

出力ファイル:

```text
data/assessments/cognitive_model_comparison_for_codex.txt
```

また、生徒AI単体評価では、パラメータの意味と観測される効果をすぐ確認できるように `Parameter Guide` を出力する。

出力ファイル:

```text
data/assessments/student_ai_evaluation_for_codex.txt
```

主に見る表:

| 表 | 確認すること |
| --- | --- |
| `Parameter Guide` | 各パラメータが何を意味し、何に影響するか |
| `Learning Curve` | 理解度・スキル習得度と正答率の関係 |
| `Difficulty Breakdown` | 難易度別に正答率・正答確率が変わるか |
| `Skill Breakdown` | 弱点化したスキルの問題で正答確率が下がるか |
| `Misconception Comparison` | 誤概念が関連問題に効いているか |

これにより、同じ問題でも次の違いを表現できる。

- 理解度・スキル習得度が高い生徒は正答しやすい
- 同じ理解度でも、問題難易度が高いと誤答しやすい
- 理解していても `slip` により間違えることがある
- 理解していなくても `guess` により正答することがある
- 該当スキルが低い生徒は誤答しやすい
- 誤概念がある生徒は関連問題で誤答しやすい
- 理解度が上がると誤概念の影響が弱まる

この設計により、理解度と正答率には相関を持たせるが、両者を同一視しない。

## 個人特徴モデル

個人特徴モデルは、JSON上の心理・性格パラメータを発話スタイルへ変換する。

| 状態 | 発話への反映 |
| --- | --- |
| 自己効力感が低い | 断定を避ける、不安を出す |
| 質問傾向が高い | 具体的に質問する |
| モチベーションが低い | 短い返答、粘りの弱さ |
| 誠実性が高い | 途中式を丁寧に出す |
| 外向性が低い | 発話が短い |
| 神経症傾向が高い | 不安や確認が増える |

この層は、正答率そのものを決めない。発話の見え方を変える。

## LLM発話生成器

LLMは、認知モデルと個人特徴モデルから作られた指示に従って、生徒らしい自然文を生成する。

LLMの責任:

- 生徒らしい返答を書く
- 性格に応じて口調を変える
- 途中式の出し方を変える
- 不安、質問、短さ、丁寧さを表現する

LLMに任せないこと:

- 正答するかどうか
- どの誤概念を持つか
- 授業によって理解度がどれだけ上がるか
- 教師AIへ渡す推定情報

発話制約:

- 生徒1ターンだけ出力する
- 教師発話を書かない
- `教師:`、`先生:`、`生徒:` などの台本形式を残さない
- 解答形式は `答え: x = ...` に統一する
- 数式を `plus` や `equals` のように英語化しない

## 教育シミュレーション内での位置づけ

教育シミュレーションでは、生徒AIの内部状態を教師AIに直接見せない。

```text
真の生徒状態
  ↓
生徒AIが授業中の反応を生成
  ↓
観察可能情報に変換
  ↓
伝達AIが個人・クラス状態を要約
  ↓
教師AIが授業構成や個別支援を考える
```

教師AIが使える情報:

- 生徒発話
- 正誤
- 反応の短さ
- 質問の有無
- 途中式の有無
- 観察から推定されたクラス傾向

教師AIが直接見てはいけない情報:

- 真の `knowledge_state`
- 真の `misconceptions`
- 真の `big_five`
- 真の `motivation`
- 真の `self_efficacy`

## 内部妥当性を検証する実験

内部妥当性は次の5観点で評価する。

| 評価観点 | 確認すること | 主な出力 |
| --- | --- | --- |
| 認知的学習曲線 | 理解度を上げると正答率が段階的に上がるか | `Learning Curve` |
| 誤概念感度 | 誤概念あり条件で関連問題の正答率が下がるか | `Misconception Comparison` |
| スキル別弱点 | 特定スキルだけ弱くすると該当問題の正答確率が下がるか | `Skill Breakdown` |
| 個人特徴の発話反映 | 自己効力感、質問傾向、意欲などが発話特徴に出るか | `Utterance Samples` |
| 生徒発話形式 | 教師発話を混ぜず、生徒1ターンとして扱えるか | `one_turn_student_response` |

現在の実装では、`student_ai_colab.ipynb` から `run_student_ai_evaluation()` を実行することで、これらの評価をまとめて出力できる。

出力ファイル:

```text
data/assessments/student_ai_evaluation_for_codex.txt
```

この出力は、実際の生徒との一致を示すものではない。あくまで、設計した内部状態操作に対して、生徒AIが一貫した反応を返すかを確認するものである。

## 汎用化について

一次方程式に限定しない汎用的な生徒AIを作ることは、不可能ではない。

ただし、完全に教科非依存の認知モデルを一つ作るのは現実的ではない。誤概念、スキル分解、問題の採点方法は単元ごとに大きく異なるためである。

現実的な設計は次である。

```text
共通基盤 + 単元別プラグイン
```

共通化できる部分:

- 生徒ID管理
- 学習履歴
- 自己効力感
- 質問傾向
- モチベーション
- Big Five
- LLM発話生成
- ログ保存
- 伝達AIによる観察要約
- 教師AIへの情報受け渡し

単元ごとに差し替える部分:

- スキル定義
- 問題データ
- 採点器
- 誤概念マップ
- 正答確率モデル
- 誤答生成ロジック
- 学習更新ルール

## 論文での主張範囲

主張してよいこと:

- 生徒AIの知識状態を外部認知モデルで制御できる
- 理解度と正答率の関係を段階的に作れる
- 誤概念の有無で関連問題の正答確率を変えられる
- 個人特徴を発話スタイルに反映できる
- 教師AIには観察可能情報だけを渡す構成にできる
- 教育シミュレーション用代理生徒としての内部妥当性を確認した

主張しないこと:

- 人間学習者を完全に再現した
- 実際の生徒集団の分布を再現した
- 実際の教育効果を証明した
- LLMが内部的に理解度や誤概念を持っている
- すべての教科・単元で同じ認知モデルが使える

## 今後の外的妥当性

外的妥当性を扱うには、実クラスのデータが必要になる。

必要なデータ例:

- 実クラスの問題別正誤
- 生徒別正答率
- スキル別正答率
- 誤答内容
- 授業中の発話や質問

将来的には、実クラスの分布とAI生徒集団の分布を比較し、平均、標準偏差、問題別正答率、スキル別正答率、誤答パターンの差分を見る。

現時点ではそのデータがないため、外的妥当性は扱わず、内部妥当性の検証に集中する。
