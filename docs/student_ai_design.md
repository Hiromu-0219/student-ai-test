# 生徒AI 設計仕様書

作成日: 2026-07-23

## 目的

本仕様書は、教育シミュレーションに組み込む生徒AIの設計を固定するための文書である。

本プロジェクトの生徒AIは、人間学習者を完全に再現するものではない。目的は、教師AIの授業設計を検証するために、理解度、誤概念、個人特徴を制御できる「限定的な代理生徒」を構築することである。

## 基本方針

生徒AIは次の3層で構成する。

```text
生徒AI = 認知モデル + 個人特徴モデル + LLM発話生成器
```

各層の役割は明確に分ける。

| 層 | 役割 | LLMに任せるか |
| --- | --- | --- |
| 認知モデル | 正答率、誤答、スキル弱点、誤概念の影響を制御する | 任せない |
| 個人特徴モデル | 自信、質問傾向、意欲、Big Fiveを発話スタイルへ変換する | 一部だけ任せる |
| LLM発話生成器 | 生徒らしい自然文を生成する | 任せる |

重要なのは、LLMに「その生徒が正解できるか」を自由に決めさせないことである。正誤や誤答傾向は外部状態と認知モデルで制御し、LLMはその結果を自然な発話にする。

## 生徒状態

生徒状態は `data/students/{student_id}.json` で管理する。

### 必須フィールド

| フィールド | 型 | 役割 |
| --- | --- | --- |
| `student_id` | string | 生徒ID |
| `name` | string | 表示名 |
| `understanding` | object | 単元ごとの大まかな理解状態 |
| `knowledge_state` | object | 単元・スキル別の理解度 |
| `error_tendency` | list | 誤答傾向 |
| `misconceptions` | list | 誤概念 |
| `learning_speed` | string | 学習の伸びやすさ |
| `personality` | object | 追加の性格記述 |
| `big_five` | object | Big Five |
| `self_efficacy` | string | 自己効力感 |
| `question_tendency` | string | 質問傾向 |
| `motivation` | string | モチベーション |
| `learning_history` | list | 学習履歴 |

### 知識状態

知識状態は0-100のスコアで扱う。

例:

```json
{
  "knowledge_state": {
    "linear_equation": {
      "score": 60,
      "can_solve_ax_plus_b_equals_c": 65,
      "can_transpose_terms": 55,
      "can_divide_by_coefficient": 45,
      "can_handle_negative_numbers": 50,
      "can_handle_fractions": 35
    }
  }
}
```

`score` は単元全体の理解度、各スキルは問題タイプごとの理解度を表す。

### 個人特徴

性格・心理系の段階評価は原則として次の5段階で扱う。

```text
very_low, low, medium, high, very_high
```

対象:

- `self_efficacy`: 自己効力感
- `question_tendency`: 質問傾向
- `motivation`: モチベーション
- `big_five.openness`
- `big_five.conscientiousness`
- `big_five.extraversion`
- `big_five.agreeableness`
- `big_five.neuroticism`

## 認知モデル

認知モデルは、生徒の真の内部状態から、問題に対する解答方針を決める。

主な出力:

| 出力 | 意味 |
| --- | --- |
| `target_correct` | この問題で正答させるか |
| `correct_probability` | 正答確率 |
| `target_answer` | 生徒AIが最終的に出す答え |
| `misconception_strength` | 誤概念の強さ |
| `active_misconceptions` | 今回の問題に効いている誤概念 |
| `rationale` | なぜその正誤になったか |

### 正答確率の考え方

正答確率は次の要素から決める。

```text
正答確率 = スキル理解度 + 単元全体理解度 + 自己効力感補正 + モチベーション補正 - 誤概念ペナルティ
```

これにより、同じ問題でも次の違いを表現できる。

- 理解度が高い生徒は正答しやすい
- 該当スキルが低い生徒は誤答しやすい
- 誤概念がある生徒は関連問題で誤答しやすい
- 理解度が上がると誤概念の影響が弱まる

## 個人特徴モデル

個人特徴モデルは、JSON上の心理・性格パラメータを発話スタイルへ変換する。

例:

| 状態 | 発話への反映 |
| --- | --- |
| 自己効力感が低い | 断定を避ける、不安を出す |
| 質問傾向が高い | 具体的に質問する |
| モチベーションが低い | 短い返答、粘りの弱さ |
| 誠実性が高い | 途中式を丁寧に出す |
| 外向性が低い | 発話が短い |
| 神経症傾向が高い | 不安や確認が増える |

この層は、正答率そのものを決めない。あくまで発話の見え方を変える。

## LLM発話生成器

LLMは、認知モデルと個人特徴モデルから作られた指示に従って、生徒らしい自然文を生成する。

LLMの責任:

- 生徒らしい返答を書く
- 性格に応じて口調を変える
- 途中式の出し方を変える
- 不安、質問、短さ、丁寧さを表現する

LLMに任せないこと:

- 正答するかどうか
- どの誤概念を持つか
- 授業によって理解度がどれだけ上がるか
- 教師AIへ渡す推定情報

発話制約:

- 生徒1ターンだけ出力する
- 教師発話を書かない
- `教師:`、`先生:`、`生徒:` などの台本形式を残さない
- 解答形式は `答え: x = ...` に統一する
- 数式を `plus` や `equals` のように英語化しない

## 教育シミュレーション内での位置づけ

教育シミュレーションでは、生徒AIの内部状態を教師AIに直接見せない。

流れ:

```text
真の生徒状態
  ↓
生徒AIが授業中の反応を生成
  ↓
観察可能情報に変換
  ↓
伝達AIが個人・クラス状態を要約
  ↓
教師AIが授業構成や個別支援を考える
```

教師AIが使える情報:

- 生徒発話
- 正誤
- 反応の短さ
- 質問の有無
- 途中式の有無
- 観察から推定されたクラス傾向

教師AIが直接見てはいけない情報:

- 真の `knowledge_state`
- 真の `misconceptions`
- 真の `big_five`
- 真の `motivation`
- 真の `self_efficacy`

## 汎用化について

一次方程式に限定しない汎用的な生徒AIを作ることは、不可能ではない。

ただし、完全に教科非依存の認知モデルを一つ作るのは現実的ではない。なぜなら、誤概念、スキル分解、問題の採点方法は単元ごとに大きく異なるからである。

現実的な設計は次である。

```text
共通基盤 + 単元別プラグイン
```

### 共通基盤

教科・単元を問わず共通化できる部分:

- 生徒ID管理
- 学習履歴
- 自己効力感
- 質問傾向
- モチベーション
- Big Five
- LLM発話生成
- ログ保存
- 伝達AIによる観察要約
- 教師AIへの情報受け渡し

### 単元別プラグイン

単元ごとに差し替えるべき部分:

- スキル定義
- 問題データ
- 採点器
- 誤概念マップ
- 正答確率モデル
- 誤答生成ロジック
- 学習更新ルール

例:

```text
domains/
  linear_equation/
    skills.json
    misconceptions.json
    test_bank.json
    grader.py
    cognitive_model.py
  linear_function/
    skills.json
    misconceptions.json
    test_bank.json
    grader.py
    cognitive_model.py
```

この形にすると、教育シミュレーションの大枠は保ったまま、数学の別単元や他教科へ拡張できる。

## 現時点のスコープ

現時点では、実装と検証のスコープを一次方程式に限定する。

理由:

- スキル分解が比較的明確
- 自動採点がしやすい
- 誤概念を設計しやすい
- 学習曲線と正答率の関係を検証しやすい
- 教育シミュレーションのMVPとして扱いやすい

ただし、設計思想は一次方程式専用に閉じない。将来的には、単元別プラグインを追加する形で拡張する。

## 論文での主張範囲

主張してよいこと:

- 生徒AIの知識状態を外部認知モデルで制御できる
- 理解度と正答率の関係を段階的に作れる
- 誤概念の有無で関連問題の正答確率を変えられる
- 個人特徴を発話スタイルに反映できる
- 教師AIには観察可能情報だけを渡す構成にできる

主張しないこと:

- 人間学習者を完全に再現した
- 実際の教育効果を証明した
- LLMが内部的に理解度や誤概念を持っている
- すべての教科・単元で同じ認知モデルが使える

## 次に固めること

1. 一次方程式ドメインのスキル定義を正式化する
2. 誤概念マップを表にする
3. 学習更新ルールを設計する
4. 複数生徒クラスでの状態分布を決める
5. 伝達AIが観察できる情報の範囲を固定する
6. 外的妥当性を確認するため、人間評価または別LLM評価の手順を作る
