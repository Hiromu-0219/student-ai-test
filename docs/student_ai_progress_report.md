# 生徒AI設計 進捗報告

作成日: 2026-07-23

## 目的

本プロジェクトでは、教育シミュレーションで使用する生徒AIを構築している。

目的は、人間学習者を完全に再現することではなく、一次方程式の学習場面において、理解度、誤概念、個人特徴を制御できる「限定的な代理生徒」を作ることである。

この生徒AIを使うことで、教師AIや講義設計AIが、複数生徒の反応を見ながら授業方略を検討できる環境を作る。

## 基本方針

生徒AIは、LLM単体にすべてを任せる構成にはしていない。

```text
生徒AI = 認知モデル + 個人特徴モデル + LLM発話生成器
```

役割分担:

| 要素 | 役割 |
| --- | --- |
| 認知モデル | 正答・誤答、正答確率、誤概念の影響を制御する |
| 個人特徴モデル | 自己効力感、質問傾向、モチベーション、Big Fiveを発話スタイルに反映する |
| LLM発話生成器 | 認知モデルと個人特徴モデルの結果を自然な生徒発話に変換する |

重要な点は、LLMに「この生徒が正解できるか」を自由に決めさせないことである。
正誤や誤答傾向は外部の認知モデルで決め、LLMは主に発話生成に使う。

## 生徒状態

生徒状態は `data/students/*.json` で管理する。

主なパラメータ:

| パラメータ | 意味 |
| --- | --- |
| `knowledge_state.linear_equation.score` | 一次方程式全体の理解度 |
| `knowledge_state.linear_equation.<skill>` | 操作・スキル別の習得度 |
| `misconceptions` | 誤概念 |
| `self_efficacy` | 自己効力感 |
| `question_tendency` | 質問傾向 |
| `motivation` | モチベーション |
| `big_five` | 性格特性 |
| `learning_history` | 学習履歴 |

理解度は単一の値だけでなく、スキル別に細分化している。
理由は、同じ理解度の生徒でも「移項はできるが係数で割る操作が苦手」「負の数で崩れる」「分数で止まる」など、つまずき方が異なるためである。

## 認知モデル

現在は2種類の認知モデルを用意している。

| モデル | 内容 |
| --- | --- |
| `legacy` | 従来の理解度中心モデル |
| `bkt_irt` | BKT/IRT寄りの認知モデル |

BKT/IRT寄りモデルでは、理解度を正答率そのものとは見なさない。

```text
理解度・スキル習得度 = 潜在的な知識状態
正答・誤答 = 知識状態、問題難易度、誤概念、guess、slipから生成される観測結果
```

これにより、以下を表現できる。

- 低理解度でも `guess` により正答することがある
- 高理解度でも `slip` により誤答することがある
- 同じ理解度でも、難しい問題ほど正答確率が下がる
- 誤概念がある場合、関連問題の正答確率が下がる

## 発話生成までの流れ

生徒AIが発話するまでの流れは次の通りである。

```text
教師が問題を出す
  ↓
生徒状態JSONを読む
  ↓
認知モデルが正答・誤答方針を決める
  ↓
個人特徴モデルが話し方を決める
  ↓
LLMまたはMockモデルが生徒発話を生成する
  ↓
ログに保存する
```

例:

```text
問題: 3x = 15
スキル: can_divide_by_coefficient
誤概念: 3を引けばよいと思っている
自己効力感: low
質問傾向: high
```

この場合、生徒AIは「3で割る」のではなく「3を引く」ような誤答を出しつつ、自信なさげな発話や確認質問を含める。

## 教育シミュレーション内での位置づけ

教師AIや講義設計AIは、生徒AIの真の内部状態を直接見ない。

全体構成:

```text
生徒AI
  ↓
伝達AI
  ↓
講義設計AI
  ↓
教師発話AI
  ↓
生徒AI
```

伝達AIは、生徒の発話、正誤、質問の有無、途中式の有無など、授業中に観察可能な情報だけを見て要約する。

講義設計AIは、その観察要約や `teacher_beliefs` を使って、クラス全体に対する講義目標、時間配分、全体方略、個別支援方針を提案する。

## 現在できていること

- 生徒状態JSONによる生徒個別パラメータ管理
- 一次方程式の20問テスト
- 理解度0-100と正答率の関係の検証
- 誤概念あり/なしの比較
- スキル別弱点の比較
- 問題難易度別の正答率・正答確率の比較
- BKT/IRT寄りモデルと従来モデルの比較
- 個人特徴による発話差の確認
- Codex/ChatGPT共有用txt出力
- 複数生徒クラスでの授業実行
- 伝達AIによる複数生徒の観察要約
- 講義設計AIによるクラス全体の講義案作成

## 評価用出力

生徒AIの内部妥当性を確認する共有用ファイル:

```text
data/assessments/student_ai_evaluation_for_codex.txt
```

従来モデルとBKT/IRT寄りモデルの比較ファイル:

```text
data/assessments/cognitive_model_comparison_for_codex.txt
```

授業方略・講義設計の共有用ファイル:

```text
data/assessments/teaching_strategy_result_summary.txt
```

## 現時点での主張範囲

主張できること:

- 生徒AIの内部状態を外部パラメータとして制御できる
- 理解度、問題難易度、スキル弱点、誤概念が正答確率に反映される
- 個人特徴が発話スタイルに反映される
- 生徒AIの真の内部状態を教師側に直接見せず、観察可能情報を通して授業設計へ接続できる

主張しないこと:

- 実際の人間生徒を完全に再現した
- 実クラスの成績分布を再現した
- BKT/IRTを実データから推定した
- LLMが内部的に理解度や誤概念を持っている
