# 論文用 実験コア整理

作成日: 2026-07-17

## 研究のコア

本研究の中心は、教師AIが生徒AIの内部状態を直接参照せず、授業中に観察できる生徒反応をもとに、クラス全体の状態を推定し、授業構成を変えられるかを検証することである。

## 研究の問い

授業中に観察可能な情報だけを使って、クラス状態に応じた授業構成を生成できるか。

## システムの流れ

```text
生徒AIの内部状態
↓
授業中に観察可能なイベントへ変換
↓
伝達AIが個人反応とクラス全体を要約
↓
教師AIがクラス状態から授業構成を生成
↓
クラス条件別に授業構成を比較
```

## 論文MVPで示すこと

- 生徒ごとに理解度、誤概念、個人特徴を持つ
- 教師AIには生徒の内部状態を直接見せない
- 観察可能な発話、正誤、反応時間などだけを扱う
- 伝達AIが複数生徒の反応をクラス要約に変換する
- 教師AIがクラス状態に応じて授業目標、時間配分、個別支援方針を変える
- クラス条件を変えたとき、授業構成が変化することを表で示す

## 今はやらないこと

- LLM発話の自然さを主成果にすること
- 現実の生徒の学習効果を証明すること
- 長い複数ターン授業を完全に再現すること
- 個別支援の効果を統計的に検証すること
- 他教科、多単元へ広げること

## 論文に使う主実験

### 実験1: クラス条件別の授業構成比較

低理解クラス、混合クラス、高理解クラスの3条件を用意し、同じカリキュラムに対して教師AIが生成する授業構成を比較する。

見る項目:

- 授業目標
- クラス平均推定理解度
- 低理解度生徒数
- 時間配分
- 個別支援方針
- 判断理由

### 実験2: 情報分離の確認

教師AIへ渡す情報に、真の `knowledge_state`、`big_five`、`motivation`、`misconceptions` などの内部状態が直接入っていないことを確認する。

見る項目:

- 観察可能イベントのキー
- 教師AIコンテキストのキー
- 内部状態フィールドが混入していないこと

## 使用する主なソースコード

| ファイル | 役割 |
| --- | --- |
| `src/class_manager.py` | クラス定義と所属生徒を読み込む |
| `src/observer/observation_filter.py` | 授業中に観察可能なイベントを作る |
| `src/observer/trait_classifier.py` | 伝達AIとして個人反応とクラス全体を要約する |
| `src/teacher/belief_manager.py` | 教師側の推定状態を更新する |
| `src/teacher/lesson_planner.py` | クラス状態から授業構成を作る |
| `data/classes/*.json` | クラス条件を定義する |
| `data/students/*.json` | 生徒AIの内部状態を定義する |
