# 2026-07-16 クラス管理追加メモ

## 目的

複数生徒を扱う授業シミュレーションで、個人状態とクラス集団の条件を分けて管理できるようにする。

## 追加した設計

- 個人状態は引き続き `data/students/{student_id}.json` に保存する
- クラス定義は `data/classes/{class_id}.json` に保存する
- `src/class_manager.py` でクラス定義、生徒一覧、クラス特徴を読み込む
- Colabでは `CLASS_ID` を変更するだけで対象クラスを切り替える

## 用意したクラス

- `class_3_basic`: 早い確認用の3人クラス
- `class_5_mixed`: 小規模な混合クラス
- `class_10_mixed`: 理解度と質問傾向がばらつく10人クラス
- `class_20_mixed`: 3-20人想定のフルクラス実験用

## クラスごとに見られる特徴

`ClassManager.summarize_class()` で以下を確認できる。

- 所属生徒数
- 所属生徒ID
- クラス定義上の特徴
- 平均理解度
- 理解度のばらつき
- 低理解度生徒
- 高理解度生徒
- 自己効力感、質問傾向、モチベーション、不安傾向の分布
- 誤概念を持つ生徒数と生徒ID

## Colabでの使い方

`notebooks/teaching_strategy_experiment.ipynb` の実験設定セルで変更する。

```python
CLASS_ID = "class_20_mixed"
```

その後、複数生徒反応のセルを実行すると、指定したクラスの全生徒を対象に反応生成、伝達AI要約、教師側推定、授業構成作成へ進められる。

## 出力

最新のクラス要約は以下にも保存する。

```text
data/assessments/class_summary_latest.json
```

## 研究上の意味

この変更により、単に生徒を増やすだけでなく、クラス条件を明示したうえで授業構成がどう変わるかを比較できる。論文では「個人モデル」と「クラス集団モデル」を分けた教育シミュレーション設計として説明しやすくなる。
