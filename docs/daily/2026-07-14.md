# 2026-07-14 進捗

## 目的

生徒AIの反応を教師AIの授業設計につなげるため、伝達AI・教師側推定・教師発話の流れを作る。

## 実装したこと

- 生徒AIの発話から個人特徴を推定する伝達AIを追加
- 伝達AIが複数生徒をまとめて要約できるようにした
- 授業中に観察可能な情報だけを扱う `observable_event` を追加
- 生徒の真の内部状態と、教師が観察から推定する状態を分離
- 教師側推定 `teacher_belief` を追加
- 複数回の授業で `teacher_belief` が蓄積される構造を作成
- 授業方略を教師発話へ変換する `RuleBasedTeacherUtteranceBuilder` を追加
- 生成した教師発話を次ターンの生徒AI反応へ接続

## 重要な設計

教師は生徒の内部パラメータを直接見ない。

代わりに、授業中に見える発話、正誤、反応時間、質問の有無などから、伝達AIが観察情報を要約し、教師側の推定状態を更新する。

```text
true_student_state
  生徒AIの内部状態。教師には直接見せない。

observable_event
  授業中に観察できる情報。

communication_ai
  観察情報を要約し、教師に渡す。

teacher_belief
  教師側に蓄積される生徒理解。
```

## 確認できたこと

- 伝達AIが個別生徒だけでなく、クラス全体を要約できる
- 複数回授業を行うと、教師側推定に履歴が蓄積される
- 個別対応と全体対応を分ける設計が必要
