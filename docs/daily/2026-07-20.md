# 2026-07-20 作業メモ

## 今日の方針

授業方略実験で、伝達AIだけでなく、生徒発話と教師発話もLLMで生成できる構成にする。

研究上は、内部状態を直接見せず、授業中に観察できる発話や反応から伝達AIが要約し、その情報をもとに教師AIが授業構成や支援方針を決める流れを確認する。

## 実装したこと

- `StudentAISimulator` に外部の発話生成器を渡せるようにした。
- `teaching_strategy_experiment.ipynb` で共有LLMを1回だけ読み込み、生徒AI・伝達AI・教師発話生成で使い回せるようにした。
- 教師発話用に `LLMTeacherUtteranceBuilder` を追加した。
- LLM教師発話が失敗した場合は、ルールベース発話にフォールバックするようにした。
- ノートブック冒頭の最終更新日時を更新した。

## 実験で確認すること

- `USE_LLM_STUDENT_UTTERANCES = True` で生徒発話がLLM生成になること。
- `USE_LLM_COMMUNICATION_AI = True` で伝達AIの分類・要約がLLM生成になること。
- `USE_LLM_TEACHER_UTTERANCES = True` で教師の全体向け発話・個別向け発話がLLM生成になること。
- `generation_mode` が `llm` になること。
- LLMのロード時間を抑えるため、検証時は軽量モデルを使い、最終確認だけ4B系モデルに切り替えること。

## 注意

Colabではランタイムを再起動するとLLMを毎回読み込み直す必要がある。ノートブック内では共有LLMを再利用するため、同一ランタイム内であれば生徒・伝達AI・教師で別々にモデルをロードしない。
