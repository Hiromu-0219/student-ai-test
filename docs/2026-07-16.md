# 2026-07-16 進捗

## 目的

研究の中心を「単発の声かけ最適化」ではなく、「クラス全体を見た講義全体の構成最適化」に置く。

## 実装したこと

- `RuleBasedLessonPlanner` を追加
- クラス全体の `teacher_belief` から `class_profile` を作成
- `class_profile` をもとに授業全体を計画
  - 授業目標
  - 導入
  - 全体説明
  - 例題
  - 個別演習
  - 確認
  - 個別支援方針
- Colabノートブックに「クラス全体を見て講義全体の構成を作成」セルを追加
- クラス状態別に授業構成が変わるかを検証するセルを追加
- `LessonSessionRunner` を追加
- 講義全体の構成に沿って、導入、全体説明、例題、個別演習、確認を複数ターンで自動実行できるようにした
- 各フェーズで、生徒反応、正誤判定、観察イベント、伝達AI要約、teacher_belief更新をまとめて記録できるようにした
- 個別演習フェーズで、特定生徒だけに個別支援メッセージを届ける分岐を追加
- Colabの授業セッション表に `individual_message_count` を追加
- 授業セッション実行時に、9セルで作成した `latest_teacher_beliefs` を初期値として引き継ぐように修正
- 比較対象として以下のクラス状態を用意
  - 低理解クラス
  - 高理解クラス
  - 理解度混在クラス
  - 質問しにくいクラス
  - 係数処理の誤概念が多いクラス
- 比較結果をCSV/JSONで保存

## 現在の到達点

現在は、次の流れが動くところまで進んでいる。

```text
複数生徒AIの反応
→ 授業中に観察できる情報へ変換
→ 伝達AIがクラス全体を要約
→ 教師側beliefを更新
→ クラス全体の状態から講義全体を設計
→ クラス全体対応と個別対応を計画
→ 教師発話へ変換
→ 次ターンの生徒反応へ接続
→ 講義全体の各フェーズを複数ターンで自動実行
```
