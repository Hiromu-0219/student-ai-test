# 先輩共有用まとめ

作成日: 2026-07-16

## 研究・開発の方向性

一次方程式を題材に、教師1人に対して複数生徒がいる授業場面をシミュレーションする生徒AIシステムを作っています。

重要な方針は、LLMに生徒の理解度や誤答を直接決めさせるのではなく、認知モデルとJSON状態で生徒を制御し、LLMは主に発話生成に使うことです。

## 現在できていること

- 複数の生徒AIを用意できる
- 生徒ごとに理解度、誤概念、性格、自己効力感、質問傾向、モチベーションなどを持たせられる
- 一次方程式の問題に対して、生徒状態に応じた反応を出せる
- 授業中に見える情報だけを `observable_event` として抽出できる
- 伝達AIが複数生徒の反応を要約できる
- 教師側に、生徒理解の推定値 `teacher_belief` を蓄積できる
- クラス全体の推定状態から、講義全体の構成を作れる
- 授業目標、時間配分、全体説明、例題、個別演習、確認、個別支援方針を出せる
- Colab上で実験できるノートブックを用意している

## 設計のポイント

教師は生徒AIの内部パラメータを直接見ません。

実際の授業に近づけるため、教師が使える情報は発話、正誤、反応時間、質問の有無などの観察可能な情報に限定しています。その情報を伝達AIが要約し、教師側の推定状態を更新します。

```text
生徒AIの内部状態
→ 授業中に観察できる情報
→ 伝達AIの要約
→ 教師側の生徒理解
→ 講義全体の構成
→ 教師の発話・支援
```

## 現在の実装状態

現在はLLMを本格投入する前の段階です。

Colabでは軽量に検証するため、主に `USE_MOCK_MODEL = True` で動かしています。生徒の知識状態や誤答傾向は認知モデル側で制御し、今後LLMを発話生成や伝達AI・教師AIの一部に入れて比較する予定です。

## 直近で確認したこと

クラス状態を変えると、授業構成が変わるかを確認するセルを追加しました。

比較したクラス状態:

- 低理解クラス
- 高理解クラス
- 理解度混在クラス
- 質問しにくいクラス
- 係数処理の誤概念が多いクラス

これにより、伝達AIがまとめた生徒情報や教師側推定によって、授業目標や時間配分が変わるかを確認できます。

## 次にやること

1. 講義全体の構成に沿って、複数ターンの授業を自動実行する
2. 個別発話を特定生徒だけに届ける分岐を作る

5. 授業後に生徒AIの真の知識状態が少しずつ変わる仕組みを接続する
