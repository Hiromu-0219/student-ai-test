# data

シミュレーションで使う状態、問題、ログ、実験結果の置き場です。

## ディレクトリ

| パス | 役割 |
| --- | --- |
| `students/` | 生徒ごとの真の内部状態。理解度、誤概念、個人特徴、学習履歴を保存 |
| `classes/` | クラス構成。3人、5人、10人、20人などのクラス設定 |
| `teacher_beliefs/` | 教師側が観察から持っている生徒理解。真の内部状態とは分けて管理 |
| `tests/` | 一次方程式のテスト問題 |
| `curriculum/` | 単元・スキル定義 |
| `observations/` | 授業中に観察されたイベント |
| `logs/` | 通常対話ログ |
| `assessments/` | 学力テスト結果、評価結果、Codex共有用txt |

## 状態管理の考え方

`students/` は生徒AIの内部状態です。実環境の教師や伝達AIが最初からすべて見られる情報ではありません。

教育シミュレーションでは、授業中に観察できる発話・正誤・反応から `teacher_beliefs/` を更新し、その情報をもとに講義設計AIが授業方針を考えます。
